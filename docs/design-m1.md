# M1 design notes (vault core)

Decisions made during implementation, ADR-lite. Each traces to a spec that left the
choice open. Full ADRs start in M2 when swarm-design exists to write them.

1. **One file, `scripts/swarmvault.py`** — query + sync + init/register + claims in a
   single ~900-line script. Install is "copy one file"; NFR-M1 said single-file scripts
   and merging beats sharing a module between two files. (FR-03/FR-04)
2. **`claim`/`release` are subcommands**, not just a documented convention. The atomic
   O_CREAT|O_EXCL dance is easy to get wrong in ad-hoc shell; one command makes every
   agent do it identically on every platform. (FR-05)
3. **Codex/platform-neutral memory lives in `<project>/.swarm/memory/`** — any agent can
   write plain markdown there without Claude-specific paths; sync mirrors it exactly like
   Claude Code memory. Claude memory source is read from `~/.claude/projects/<enc>/memory/`
   as before. (FR-04 "Codex equivalents when present")
4. **Legacy `claude/*` tags accepted** alongside canonical `swarm/*` in type
   classification, so an existing Claude Vault can migrate without a retag sweep. (FR-01)
5. **Lock = lock file with 600 s staleness**, not flock(2) — O_CREAT|O_EXCL is portable
   (Windows included); flock is not. Stale locks are broken with a stderr note. (FR-04)
6. **Sessions mirror is Claude-Code-only in v1**; Codex agents journal directly into
   `40 Sessions/` (own-file writes make that safe). Hand-journaled session notes are
   listed in MOCs alongside mirrored ones. (FR-04)
7. **Context budget softness implemented as whole-section trimming**: the recent-sessions
   line is dropped first; memory/digest/phase essentials are never dropped even when over
   budget. (NFR-P2 as amended)
8. **`vault-template/` is generated by `init`** (committed output, `.gitkeep` for empty
   dirs), so template-vs-code drift is impossible by construction. (FR-01)
